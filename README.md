## Access Management Model
User access and repository permissions are managed **strictly through teams** to ensure proper segregation of duties and minimize unauthorized access.

### Core Teams
The following standard teams will be provisioned for each onboarding request:

- **Admin Team**
  - Full administrative access to associated repositories

- **Write Team**
  - Write (push) access to designated repositories

- **Read Team**
  - Read-only access to designated repositories

Access is granted based on **team membership only**.

---

## Getting Started
To onboard your team or repository into the **Comcast-Enterprise GitHub Organization**:

1. Submit a request through the designated onboarding queue
2. The platform team will provision teams and repositories
3. Access will be granted based on the approved request

Onboarding is handled promptly once the request is submitted.

---

## Alternative Options
Teams may continue using one of the **existing GitHub organizations** if it better aligns with their operational or ownership requirements.
